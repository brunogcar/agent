# 📈 B3 (Brasil, Bolsa, Balcão)

Brazilian stock exchange data via the new paginated JSON API.

## Sub-domains

| Sub-domain | What | Storage | Source |
|---|---|---|---|
| [api](api/) | Market data: instruments (tickers, ISIN, company names), trades (prices, volume), derivatives | `memory_db/b3/{table}.db` | `arquivos.b3.com.br/tabelas/table/` |

## API Migration (2026)

B3 migrated from the old 3-step CSV download flow (publications → token → CSV) to a React SPA with a paginated JSON API. The old token endpoint (`/api/download/requestname`) returns HTTP 400 for all server-side requests. The new API works without authentication:

```
GET /tabelas/table/{tableName}/{date}/{page}
→ {"name", "columns": [...], "values": [[...], ...], "pageCount": N}
```

20 rows per page, JSON format, no token needed.

## Tables

| Table | API Name | Key | What |
|---|---|---|---|
| instruments | InstrumentsConsolidated | TckrSymb | All listed securities: ticker, ISIN, company name, segment, governance |
| trades | TradeInformationConsolidated | TckrSymb+RptDt | Daily regular session: prices, volume, VWAP |
| after_hours | TradeInformationConsolidatedAfterHours | TckrSymb+RptDt | After-hours session trades |
| derivatives | DerivativesOpenPosition | TckrSymb | Open interest: futures, options |

## Bridge Integration

The `instruments` table has `TckrSymb` (ticker) + `CrpnNm` (company name). The CVM `cad` data source has `CNPJ` + `CD_CVM` + `DENOM_SOCIAL`. The bridge joins them: ticker → CrpnNm → DENOM_SOCIAL → CNPJ → CD_CVM → financial statements.
