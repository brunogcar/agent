In 2.0, consider splitting into separate `node_push`, `node_pr_create`, `node_pr_merge` for finer-grained routing and retry. **[v2.0-rc1] RESOLVED** in v2.0-beta Phase 3.3 — `node_publish` split into `node_push` + `node_create_pr` + `node_merge_pr`; original kept as backward-compat wrapper. |
